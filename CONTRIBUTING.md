# 贡献指南

感谢您对 kn-fetch 项目的关注！

## 如何贡献

### 报告 Bug

在提交 Bug 前，请先搜索 [Issues](../../issues) 确认是否已有人报告。

报告 Bug 时请提供：

1. 详细的 Bug 描述
2. 复现步骤
3. 期望行为 vs 实际行为
4. 环境信息（OS、Python 版本等）
5. 相关的日志输出

### 提出新功能

1. 先在 [Issues](../../issues) 中讨论，避免重复劳动
2. 提供功能的使用场景和预期效果
3. 说明是否愿意自己实现

### 提交代码

#### 开发流程

1. Fork 本仓库
2. 创建特性分支：`git checkout -b feature/amazing-feature`
3. 提交更改：`git commit -m 'feat: 添加新功能'`
4. 推送到分支：`git push origin feature/amazing-feature`
5. 提交 Pull Request

#### PR 模板

```markdown
## 描述
简要描述这个 PR 的目的

## 变更类型
- [ ] Bug 修复
- [ ] 新功能
- [ ] 文档更新
- [ ] 代码重构
- [ ] 性能优化

## 测试
- [ ] 已添加测试
- [ ] 测试通过
- [ ] 已更新文档

## 检查清单
- [ ] 代码遵循项目规范
- [ ] 已更新相关文档
- [ ] 无新增警告
```

#### 代码审查

提交 PR 后，维护者会进行审查。请耐心等待并积极回应反馈。

## 开发规范

详见 [开发指南](docs/DEVELOPMENT.md)

- 遵循 PEP 8 规范
- 编写清晰的文档字符串
- 添加必要的测试
- 保持提交信息简洁明了

## 行为准则

- 尊重所有贡献者
- 建设性的讨论
- 关注问题本身，而非个人
- 接受不同的观点

## 联系方式

- GitHub Issues: [提交问题](../../issues)
- Discussions: [参与讨论](../../discussions)

感谢您的贡献！
