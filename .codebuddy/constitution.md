# 项目最高宪法（Constitution）
所有智能体必须100%遵守此宪法，任何违反此宪法的代码生成行为均视为无效。

## 第一原则：契约先行（Interface-First）
1.  **禁止跳过设计直接编码**：在生成第一行代码前，必须先输出并锁定以下3份文档，未经用户明确回复「确认契约」，不得进入编码阶段：
    - `arch.md`：架构设计（文件树、技术栈、数据库Schema）
    - `api.md`：接口契约（URL、Method、Request/Response JSON Schema）
    - `route.md`：页面路由与状态流转
2.  **契约神圣不可侵犯**：编码阶段必须严格按照锁定的契约开发，禁止私自修改接口字段、类型或请求方式。

## 第二原则：最小闭环优先（MVP Loop）
开发顺序必须严格遵循：
1.  先写后端核心接口（含Mock数据）
2.  再写前端核心调用逻辑
3.  **必须先完成一次前后端联调通**，确保系统可运行
4.  最后才允许开发非核心功能、优化UI

## 第三原则：自动化验收（No Test, No Delivery）
代码开发完成后，必须按以下顺序执行全量检查，**任何一步不通过，必须自动修复并重新执行全流程**：
1.  依赖安装检查（`npm install` 无报错）
2.  静态检查（TS类型检查、ESLint检查、import路径检查）
3.  编译检查（项目构建无报错）
4.  服务启动检查（沙箱中启动服务，无运行时异常）
5.  E2E联调测试（模拟用户完整操作流程，所有核心功能互通）

## 第四原则：交付标准（Definition of Done）
只有满足以下所有条件，才可视为项目完成：
1.  所有需求功能100%实现，无遗漏
2.  前后端接口完全互通，无404/500报错
3.  项目可一键启动，用户仅需执行安装+启动命令
4.  无控制台报错、无运行时异常
5.  包含完整的 `README.md`（启动说明、功能介绍）
